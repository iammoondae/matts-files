1

Create a Google Account (if needed)

You need a Google account to access Firebase Console

If you already have a Gmail or Google account, skip to Step 2. Otherwise:

  1. Go to accounts.google.com/signup
  2. Create a free Google account
  3. This is the account that will own your Firebase project — it doesn't have to be the same as your users' accounts
💡
Tip Use a dedicated account for your app projects if you prefer to keep it separate from your personal Gmail.
2

Create a Firebase Project

Firebase is Google's free backend service — this takes about 3 minutes

🔥 Open Firebase Console
  1. Click "Create a project" (or "Add project")
  2. Enter a project name: Matts Learning Hub (or whatever you like — this is just for your dashboard, users won't see it)
  3. It will ask about Google Analytics:
    • You can disable it for now (keep things simple) — you can enable it later
  4. Click "Create project" and wait ~30 seconds
  5. Click "Continue" when it's ready
✅
Free Tier (Spark Plan) Firebase will automatically use the free Spark plan. No credit card required. The free tier is very generous for small apps.
3

Tell Firebase about your app so they can generate the config file

  1. In your new Firebase project, click the Android icon (🤖) to add an Android app
  2. Fill in the form:
    • Android package name: dev.matteo.learninghub
    • App nickname: Matteo's Learning Hub (optional, just for your reference)
    • Debug signing certificate SHA-1:

  1. Click "Register app"
4

Download google-services.json

This file connects your app to your Firebase project

  1. After registering, Firebase will show a "Download google-services.json" button — click it
  2. Save the file
  3. You will need to place it in your project at this location:
📁
Where to put the file android-project/app/google-services.json Place it inside the app/ folder — NOT the project root, and NOT inside src/
  1. Click "Next" on the remaining Firebase setup screens (we'll handle the Gradle setup through code later — just skip through)
  2. Click "Continue to console"
⚠️
Important Do NOT share google-services.json publicly (e.g., on GitHub). It contains your project's API keys. It's safe inside your APK but shouldn't be in public repos.
5

Enable Google Sign-In Provider

Tell Firebase to accept Google accounts for login

  1. In Firebase Console, click "Authentication" in the left sidebar (under "Build")
  2. Click "Get started" if this is your first time
  3. Go to the "Sign-in method" tab
  4. Click on "Google" in the providers list
  5. Toggle the "Enable" switch ON
  6. Fill in the "Project support email" — select your Google account email from the dropdown
  7. Click "Save"
✅
That's it for authentication setup! Google Sign-In is now enabled. Your users will be able to log in with any Google account.
6

Create a Cloud Firestore Database

This is where your users' data (progress, records, settings) will be stored

  1. In Firebase Console, click "Firestore Database" in the left sidebar (under "Build")
  2. Click "Create database"
  3. Choose a location for your database:
    • Pick the one closest to your users
    • For the Philippines: choose asia-southeast1 (Singapore) or asia-east1 (Taiwan)
  4. For Security Rules, select "Start in test mode"
    • This allows open read/write for 30 days — fine for development
    • We'll set proper security rules after implementation
  5. Click "Create"
⏰
Test Mode Expires in 30 Days Test mode is temporary. After I implement the login code, we'll add proper security rules so only logged-in users can access their own data. Don't forget this step before going to production!
