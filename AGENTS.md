# Matteo's Learning Hub - Developer Subagent Roster (AGENTS.md)

This roster defines the roles and coordination protocol for specialized AI subagents in this workspace.

---

## 👥 1. Subagent Roles

### 🔍 1. Research Agent (`research`)
* **Role**: Explore codebase references, crawl web resources, and inspect compiled assets.
* **Scope**: Read-only tools. Does not write or edit any source files.
* **Usage**: Delegate long research operations (e.g. searching historical logs, browsing API documentation) to keep the parent context lean.

### 💻 2. Coder Agent (`self` or specialized `coder`)
* **Role**: Implement code updates, refactor modular JS files, and handle bug fixes.
* **Scope**: Write and edit permissions on source files (`index.html`, `index.css`, `app.js`, `modules/*.js`, `scratch/build_apk.sh`).
* **Usage**: Tasked with writing code blocks, performing replacements, and ensuring logical alignment.

### 🎨 3. UI/UX Designer Agent (`designer`)
* **Role**: Redesign layouts, styles, and animation keyframes.
* **Scope**: Write permissions on `index.css` and style segments of `index.html`.
* **Usage**: Focuses on HSL visual color systems, starry sky/warm cream themes, responsiveness, and mascot Tobi's CSS animations.

### 🧪 4. Database & Quality Validator Agent (`validator`)
* **Role**: Run quality control testing gates.
* **Scope**: Write permissions on logs, read permissions on JSON/databases, execute permissions on verification scripts.
* **Usage**: Runs `unique.py`, `validate_db.py`, and inspects generated diagrams for text wrap overflows.

---

## 🔄 2. Handoff & Coordination Protocol
* **Task Isolation**: Only one subagent should edit a specific file at any given time to prevent write conflicts.
* **Verification**: Code generated by a subagent must be verified by running the build script (`build_apk.sh`) and database validation gates before being reported as complete.
* **Lessons Logging**: If a subagent encounters a compiler/syntax error, it must document the correction pattern in `memory/lessons.md`.
